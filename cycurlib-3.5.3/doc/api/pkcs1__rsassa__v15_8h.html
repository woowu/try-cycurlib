Detailed Description

PKCS#1 v2.2 RSASSA-PKCS1-v1_5 signature scheme.

Implementation of RSASSA-PKCS1-v1_5 verification and generation. Byte format of all long numbers is Big-Endian.

See also
www.emc.com/collateral/white-papers/h11300-pkcs-1v2-2-rsa-cryptography-standard-wp.pdf

Esc_UINT8  digestType 
)


Signs a message according to RSASSA-PKCS1-v1_5_Sign.

The RSA Key Length is configured in rsa_cfg.h.

See also
PKCS#1 v2.1, section 8.2.1
rsa_cfg.h
Parameters
[in]digestDigest of the message to be signed. The length depends on the digestType parameter. See also pkcs1.h for the possible digest lengths.
[in]privKeyPrivate key structure.
[out]signatureBuffer to store the computed signature.
[in,out]signatureLenIn: Length of buffer provided by parameter signature. The buffer should be as large as the key size of the public key (e.g. for RSA-1024, it should be 1024 / 8 = 128 bytes). Out: Number of bytes written to the buffer.
[in]digestTypeThe type of the digest. One of the EscPkcs1RsaSsaV15_DIGEST_TYPE_xxx defines.
Returns
Esc_NO_ERROR if everything works fine.

Signs a message according to RSASSA-PKCS1-v1_5_Sign.

The RSA Key Length is configured in rsa_cfg.h.

Warning
This function does NOT create deep copies of any pointer passed to it! Therefore, the data behind each pointer must remain allocated/unchanged until the computation is finished!
See also
PKCS#1 v2.1, section 8.2.1
rsa_cfg.h
Parameters
[in]ctxThe context structure which stores the state of the signature generation.
[in]digestDigest of the message to be signed. The length depends on the digestType parameter. See also pkcs1.h for the possible digest lengths.
[in]privKeyPrivate key structure.
[out]signatureBuffer to store the computed signature.
[in,out]signatureLenIn: Length of buffer provided by parameter signature. The buffer should be as large as the key size of the public key (e.g. for RSA-1024, it should be 1024 / 8 = 128 bytes). Out: Number of bytes written to the buffer. This is written at the end of the computation when EscPkcs1RsaSsaV15_SignRun() finished.
[in]digestTypeThe type of the digest. One of the EscPkcs1RsaSsaV15_DIGEST_TYPE_xxx defines (see pkcs1.h).
Returns
Esc_NO_ERROR if everything works fine.

◆ EscPkcs1RsaSsaV15_SignRun()

Esc_ERROR EscPkcs1RsaSsaV15_SignRun ( EscPkcs1RsaSsaV15_SignContext *  ctx)


Run signature generation.

Each call may only compute a small part of the overall computation. Consequently, this function needs to be called until it returns an error code different from Esc_AGAIN.

Parameters
[in]ctxThe context structure which stores the state of the signature generation.
Returns
Any of the below return codes on success or a corresponding error code on failure.
Return values
Esc_AGAINif this function needs to be called again to continue signature generation.
Esc_NO_ERRORif signature generation finished successfully. The signature was written to the buffer passed to EscPkcs1RsaSsaV15_SignInit().

Verifies the signature of a message according to RSASSA-PKCS1-v1_5_Verify.

The RSA key length is configured in rsa_cfg.h.

See also
PKCS#1 v2.1, section 8.2.2
rsa_cfg.h
Parameters
[in]signatureSignature created with RSASSA-PKCS1-v1_5_Sign.
[in]signatureLenLength of the signature in bytes. This must match the modulus size of the given private key (e.g. for RSA-1024, a length of 1024 / 8 = 128 bytes).
[in]pubKeyThe public RSA key.
[in]digestDigest of the message to be verified. The length depends on the digestType parameter. See also pkcs1.h for the possible digest lengths.
[in]digestTypeThe type of the digest. One of the EscPkcs1RsaSsaV15_DIGEST_TYPE_xxx defines.
Returns
Esc_NO_ERROR if everything works fine or Esc_INVALID_SIGNATURE if verification fails.

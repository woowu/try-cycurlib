Detailed Description

EdDSA signature scheme.

EdDSA is a Schnorr signature scheme by Bernstein et al. using a twisted Edwards curve.

This implementation currently supports ed25519 variants with SHA-512.

See also
Bernstein et al. - High-speed high-security signatures (2012) RFC 8032 - Edwards-Curve Digital Signature Algorithm (EdDSA) draft-irtf-cfrg-eddsa-05 - RFC 8032 Draft

This function generates the EdDSA signature for a given message.

Description about the different ed25519 variants and their usage:

ed25519 as defined in the Bernstein paper (PureEdDSA), RFC 8032 or the RFC draft choose:

preHashed: FALSE context: Esc_NULL_PTR contextLen: 0 message: original message, non-Hashed

ed25519ctx as defined in RFC 8032

preHashed: FALSE context: Buffer with context contextLen: Length of context string, between 1 and 255 bytes. message: original message, non-Hashed

ed25519ph as defined in RFC 8032

preHashed: TRUE context: Buffer with context or Esc_NULL_PTR contextLen: Length of context string, between 0 and 255 bytes. message: SHA512 digest of the original message (hash the message before signing)

ed25519ph as defined in RFC Draft v05, also called Hashed-EdDSA in Bernstein papers

preHashed: FALSE context: Esc_NULL_PTR contextLen: 0 message: SHA512 digest of the original message (hash the message before signing)

The of usage of a context string should be carefully evaluated, cf. RFC 8032, Section 8.3.

Parameters
[in]preHashedBoolean flag, TRUE if a PreHash variant of RFC8032, e.g. ed25519ph, is used.
[out]signatureThe generated EdDSA signature.
[in,out]signatureLenIn: Size of the signature buffer in bytes. EscEdDsa_SIGNATURE_BYTES_MAX can be used as the maximum size allowed by configuration. Out: Size of the signature in bytes
[in]messageMessage to be signed.
[in]messageLenLength of the message in bytes.
[in]privKeyThe private key.
[in]privKeyLenLength of the private key in bytes. Length must match with the selected curve.
[in]pubKeyThe public key.
[in]pubKeyLenLength of the private key in bytes. Length must match with the selected curve.
[in]digestTypeThe type of the digest. Currently only EscEdDsa_DIGEST_TYPE_SHA512 is supported.
[in]curveIdThe ID of the selected Edwards curve. Currently only EscEdDsa_CURVE_ED25519 is supported
[in]contextBuffer which contains a context string. Should be set to Esc_NULL_PTR if contextLen == 0.
[in]contextLenLength of the context string. Length between 0 and 255 bytes are allowed.
Returns
Esc_NO_ERROR if everything works fine.

This function verifies the corresponding signature for a given message.

ed25519 as defined in the Bernstein paper (PureEdDSA), RFC 8032 or the RFC draft choose:

preHashed: FALSE context: Esc_NULL_PTR contextLen: 0 message: original message, non-Hashed

ed25519ctx as defined in RFC 8032

preHashed: FALSE context: Buffer with context contextLen: Length of context string, between 1 and 255 bytes. message: original message, non-Hashed

ed25519ph as defined in RFC 8032

preHashed: TRUE context: Buffer with context or Esc_NULL_PTR contextLen: Length of context string, between 0 and 255 bytes. message: SHA512 digest of the original message (hash the message before signing)

ed25519ph as defined in RFC Draft v05, also called Hashed-EdDSA in Bernstein papers

preHashed: FALSE context: Esc_NULL_PTR contextLen: 0 message: SHA512 digest of the original message (hash the message before signing)

The of usage of a context string should be carefully evaluated, cf. RFC 8032, Section 8.3.

Parameters
[in]preHashedBoolean flag, TRUE if a PreHash variant of RFC8032, e.g. ed25519ph, is used.
[in]messageThe signed message.
[in]messageLenLength of the message in bytes.
[in]signatureEdDSA signature.
[in]signatureLenLength of the signature in bytes. Length must match with the selected curve.
[in]pubKeyThe public key.
[in]pubKeyLenLength of the public key in bytes. Length must match with the selected curve.
[in]digestTypeThe type of the digest. Currently only EscEdDsa_DIGEST_TYPE_SHA512 is supported.

Any of the below return codes or a corresponding error code on failure.

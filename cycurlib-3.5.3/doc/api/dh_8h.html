Only the bit lengths 1024, 2048 and 3072 are supported.Window size for the Sliding Window algorithm Only used if the sliding window algorithm is enabled (EscDh_USE_SLIDING_WINDOW == 1). The higher the window size, the higher the RAM usage!

)

This function initializes a Diffie-Hellman context with the domain parameters p and g.

Parameters
[out]ctxDiffie-Hellman context.
[in]pPrime modulus p. The two most significant bytes must not both be zero!
[in]pLengthLength of modulus p in bytes. Must be either 128 (=1024 bit), 256 (=2048 bit) or 384 (=3072 bit)
[in]gGenerator g. Its value must be in the range [2, p-2].
[in]gLengthLength of generator g in bytes. Must be in the range [1, pLength].
Returns
Esc_NO_ERROR if everything works fine.

This function generates a random private and the corresponding public key.

Parameters
[in]ctxDiffie-Hellman context, defining the domain parameters.
[in]prngFuncPRNG function as specified in random.h. The PRNG must have been seeded before it is passed to this function!
[in]prngStateState of the PRNG function. This may be NULL if the PRNG is stateless.
[out]publicKeyThe generated public key will be stored in this struct.
[out]privateKeyThe generated private key will be stored in this struct.
Returns
Esc_NO_ERROR if everything works fine.
Esc_KEY_GENERATION_FAILED if the random numbers led to invalid private/public keys. This error is recoverable if the PRNG is sane.

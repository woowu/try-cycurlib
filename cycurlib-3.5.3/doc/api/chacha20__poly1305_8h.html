)

This function decrypts and verifies the given ciphertext and message tag using CHACHA20_POLY1305 according to RFC-7539.

In-place decryption is supported (i.e. ciphertext is overwritten with plaintext if the same array is passed for both).

Parameters
[in]keyPointer to 32 byte / 256 bit chacha20 key.
[in]noncePointer to 12 byte / 96 bit nonce.
[in]aadPointer to byte array containing the additional authenticated data.
[in]aadLenAdditional authenticated data length in bytes.
[in]cipherPointer to byte array containing the ciphertext.
[in]lengthCiphertext length in bytes.
[out]plainPointer to memory allocated for ChaCha20 decrypted plaintext, same size as ciphertext.
[in]tagPointer to 16 byte / 128 bit message tag.
Returns
Esc_NO_ERROR if everything works fine.
Esc_NULL_POINTER_ERROR if an input parameter is NULL.
Esc_INVALID_MAC if the message tag verification failed.

This function calculates ciphertext and message tag using CHACHA20_POLY1305 according to RFC-7539.

In-place encryption is supported (i.e. plaintext is overwritten with ciphertext if the same array is passed for both).

Parameters
[in]keyPointer to 32 byte / 256 bit chacha20 key.
[in]noncePointer to 12 byte / 96 bit nonce. MUST be unique for every call with the same key! Therefore it MUST NOT be randomly generated, instead a counter is recommended here.
[in]aadPointer to byte array containing the additional authenticated data.
[in]aadLenAdditional authenticated data length in bytes.
[in]plainPointer to byte array containing the plaintext.
[in]lengthPlaintext length in bytes.
[out]cipherPointer to memory allocated for ChaCha20 encrypted ciphertext, same size as plaintext.
[out]tagPointer to memory allocated for 16 byte / 128 bit Poly1305 calculated message tag.
Returns
Esc_NO_ERROR if everything works fine.
Esc_NULL_POINTER_ERROR if an input parameter is NULL.

Updates associated data (i.e.

data that is authenticated, but not encrypted). If associated data is available this function must be called after EscChaCha20Poly1305_startEncryptDecrypt, until all associated data is processed. Calling this function is optional, but it cannot be called once the encryption/decryption started (EscChaCha20Poly1305_Encrypt_Update or EscChaCha20Poly1305_Decrypt_Update was called).

Parameters

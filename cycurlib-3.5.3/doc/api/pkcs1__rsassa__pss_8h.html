Detailed Description

PKCS#1 v2.2 RSASSA-PSS signature signature scheme with MGF1 as the mask generation function.

Byte format of all long numbers is Big-Endian.

See also
www.emc.com/collateral/white-papers/h11300-pkcs-1v2-2-rsa-cryptography-standard-wp.pdf

Signs a message according to RSASSA-PSS-Sign.

See also
Section 8.1.1
Parameters
[in]prngFuncRandom function as specified in random.h
[in]prngStateRandom state which may be required by your random implementation.
[in]digestDigest of the message to be signed. The length depends on the digestType parameter. See also pkcs1.h for the possible digest lengths.
[in]privKeyPrivate key structure.
[out]signatureBuffer to store the computed signature.
[in,out]signatureLenIn: Length of buffer provided by parameter signature. The buffer must be at least as large as the key size of the public key (e.g. for RSA-1024, it should be 1024 / 8 = 128 bytes). Out: Number of bytes written to the buffer.
[in]saltLenThe length of the random salt. Maximum is given through (size of modulus - digest length - 2). The recommended value is the digest length (e.g. EscShaXXX_DIGEST_LEN).
[in]digestTypeThe type of the digest. One of the EscPkcs1_DIGEST_TYPE_XXX defines from pkcs1.h.
Returns
Esc_NO_ERROR if everything works fine

◆ EscPkcs1RsaSsaPss_SignRun()

Esc_ERROR EscPkcs1RsaSsaPss_SignRun ( EscPkcs1RsaSsaPss_SignContext *  ctx)


Run signature generation.

Each call may only compute a small part of the overall computation. Consequently, this function needs to be called until it returns an error code different from Esc_AGAIN.

Parameters
[in]ctxThe context structure which stores the state of the signature generation.
Returns
Any of the below return codes on success or a corresponding error code on failure.
Return values
Esc_AGAINif this function needs to be called again to continue signature generation.
Esc_NO_ERRORif signature generation finished successfully. The signature was written to the buffer passed to EscPkcs1RsaSsaPss_SignInit().

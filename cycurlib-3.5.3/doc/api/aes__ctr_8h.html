Applies AES CTR mode to the data.

The same function can be applied for encryption and decryption.

For encryption, source is the plaintext and dest the resulting ciphertext.

For decryption, source is the ciphertext and dest the resulting plaintext. Decrypts/Encrypts arbitrary length data in CTR mode and updates the AES context.

The context must be initialized with EscAesCtr_Init.

Parameters
[in]ctxThe previous AES CTR context.
[out]ctxThe updated AES CTR context.
[in]sourcePlaintext for encryption, ciphertext for encryption.
[out]destCiphertext for encryption, plaintext for decryption.
[in]lengthLength of data to be encrypted/decrypted in bytes.
Returns
Esc_NO_ERROR if everything works fine.

Initializes the CTR Context with the corresponding AES key.

The form of the IV is nonce|counter.

The nonce consists of the IVs bytes at indices 0..(nonceLength-1). The IV bytes at indices nonceLength..(EscAes_IV_BYTES-1) set the initial counter value having the least significant byte at position EscAes_IV_BYTES-1.

Parameters
[out]ctxThe AES CTR context.
[in]keyBitsBit length of the AES key. Can be 128, 192 or 256.
[in]keyKey bytes array. The size of the key depends on the keyBits parameter: 128 bits => 16 bytes 192 bits => 24 bytes 256 bits => 32 bytes
[in]ivIV consisting of the nonce and the initial counter value. Length must be EscAes_IV_BYTES.
[in]nonceLengthThe length of the nonce in byte. Can be 0..EscAes_IV_BYTES-1.
Returns
Esc_NO_ERROR if everything works fine.

Calculate CMAC based on DES3 for the input data

Parameters
[in]k1TDEA K1. Key bytes array. Array of size EscDes_KEY_BYTES.
[in]k2TDEA K2. Key bytes array. Array of size EscDes_KEY_BYTES.
[in]k3TDEA K3. Key bytes array. Array of size EscDes_KEY_BYTES.
[in]messageInput message which should be maced.
[in]messageLengthNumber of bytes of the input message.
[out]macBuffer to store CMAC value of macLength bytes. Can be the same as the message buffer.
[in]macLengthThe desired length of the resulting MAC, in bytes. This parameter can be used to truncate the MAC such that only the first bytes are copied to the output buffer. Must not be zero and must not be larger than EscCmacDes3_MAX_MAC_LENGTH.
Returns
Esc_NO_ERROR if everything works fine.

Initializes the CMAC context with the corresponding DES3 key.

This function does not check the key for weaknesses nor does it check the parity bits.

To check if a key is a known weak key, use the function EscDesKeytest_IsKeyWeak().

See also
des_keytest.h

Start a new MAC computation using the key provided in EscCmacDes3_Init().

This function resets the internal state such that a new MAC can be computed after EscCmacDes3_Finish(). Calling this function after EscCmacDes3_Init() is not required.

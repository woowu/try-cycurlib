Decrypts one or more data blocks and check the appended MAC for validity.

Differently from the NIST SP the encrypted plaintext is written to the plainText array even if the MAC is not valid. Nevertheless, in such a case the error Esc_INVALID_MAC is returned.

Parameters
[in]keyBitsBit length of the AES key. Can be 128, 192 or 256.
[in]keyKey bytes array. The size of the key depends on the keyBits parameter: 128 bits => 16 bytes 192 bits => 24 bytes 256 bits => 32 bytes
[in]nonceThe nonce used in the decryption.
[in]nonceLenLength of nonce in bytes.
[in]aadAdditional authenticated data.
[in]aadLenLength of aad array in bytes.
[in]cipherCiphertext array.
[in]cipherLenLength of cipherText array in bytes.
[in]tagLenLength of the tag in bytes. Can take the following values: 4, 6, 8, 10, 12, 14, 16.
[out]plainArray to store plaintext to. The array has to be at least (cipherLen - tagLen) bytes.
Note
The pointers plainText and cipherText may be the same.
Returns
Esc_NO_ERROR if everything works fine or invalid MAC.

Encrypts one or more data blocks.

Parameters
[in]keyBitsBit length of the AES key. Can be 128, 192 or 256.
[in]keyKey bytes array. The size of the key depends on the keyBits parameter: 128 bits => 16 bytes 192 bits => 24 bytes 256 bits => 32 bytes
[in]nonceThe nonce used in the decryption.
[in]nonceLenLength of nonce in bytes.
[in]aadAdditional authenticated data.
[in]aadLenLength of aad array in bytes.
[in]plainPlaintext array.
[in]plainLenLength of plainText array in bytes. The plainLen can be presented in q bytes, where q can take the following values: 2, 3, 4. The maximum of q + nonceLen is 15.
[in]tagLenLength of the tag in bytes. Can take the following values: 4, 6, 8, 10, 12, 14, 16.
[out]cipherArray to store ciphertext and tag to. The array has to be at least (plaintextLen + tagLen) bytes.
Note
The pointers plainText and cipherText may be the same.
Returns
Esc_NO_ERROR if everything works fine.

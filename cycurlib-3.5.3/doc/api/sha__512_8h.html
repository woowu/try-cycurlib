Detailed Description

SHA-384 and SHA-512 implementation. A maximum of 2^32-2 Bytes can be hashed.

See also
FIPS-180-2 at csrc.nist.gov/publications/fips/fips180-2/fips180-2withchangenotice.pdf

Esc_ERROR EscSha512_Calc ( Esc_BOOL  isSha384,
const Esc_UINT8  message[],
Esc_UINT32  messageLength,
Esc_UINT8  digest[],
Esc_UINT8  digestLength 
)


This function calculates and returns a SHA digest.

See also
FIPS-180-2
Parameters
[in]isSha384If TRUE SHA-384 is used, if FALSE SHA-512 is used.
[in]messagePointer to input data. May be a NULL pointer if messageLength is 0.
[in]messageLengthNumber of bytes of input data.
[out]digestBuffer to store digest value of digestLength bytes. Can be the same as the message buffer.
[in]digestLengthThe desired length of the digest to be stored in the buffer "digest". This parameter can be used to truncate the digest such that only the first bytes are copied to the output buffer. The value must not be zero and must not exceed the maximum digest size (either EscSha384_DIGEST_LEN or EscSha512_DIGEST_LEN depending on which sha-version is selected with isSha224).
Returns
Esc_NO_ERROR if everything works fine.

Parameters

[out]digestBuffer to store digest value of digestLength bytes.

Detailed Description

AES-GCM implementation (Galois/Counter Mode)

See also
NIST Special Publication 800-38D csrc.nist.gov/publications/nistpubs/800-38D/SP-800-38D.pdf
_aes.h

The following papers and standards are referenced:

  • The Galois/Counter Mode of Operation (GCM), David A. McGrew and John Viega

Updates associated data (i.e.

data that is authenticated, but not encrypted). If associated data is available this function must be called after EscAesGcm_startEncryptDecrypt, until all associated data is processed. Calling this function is optional, but it cannot be called once the encryption/decryption started (EscAesGcm_Encrypt_Update or EscAesGcm_Decrypt_Update was called).

Parameters
[in,out]ctxAES-GCM context
[in]aadThe next block of associated data
[in]aadLenLength of associated data block in byte
Returns
Esc_NO_ERROR if everything works fine.

Decrypts one or more data blocks (convenience function to execute all steps at once).

Parameters
[in]keyBitsBit length of the AES key. Can be 128, 192 or 256.
[in]keyKey bytes array. The size of the key depends on the keyBits parameter: 128 bits => 16 bytes 192 bits => 24 bytes 256 bits => 32 bytes
[in]ivInitialization vector.
[in]ivLenLength of the initialization vector in bytes. Must be non-zero. NIST recommends to use at most 96 bits for the IV, but the implementation does not enforce this.
[in]aadAdditional authenticated data.
[in]aadLenLength of aad array in bytes.
[in]cipherCiphertext array.
[in]cipherLenLength of cipherText array in bytes.
[out]plainArray to write decrypted ciphertext to. Has to be the same size of the cipherText array.
[in]tagAuthentication tag to verify.
[in]tagLenLength of authentication tag. Conforming to NIST 800-38D, only the values 16, 15, 14, 13 and 12 are allowed. For certain applications, the tag length may be 8 or 4.
Note
The pointers plainText and cipherText may be the same.
Returns
Esc_NO_ERROR if everything works fine.
Return values
Esc_INVALID_MACif the given tag does not match the computed one.

const Esc_UINT8  iv[],

Start a new encryption/decryption.

The context must have been initialized before calling this function. If this function was executed successfully, the caller can continue with processing additional authenticated data (EscAesGcm_AssociatedData_Update) or with encrypting/decrypting data (EscAesGcm_Encrypt_Update/EscAesGcm_Decrypt_Update).

Parameters
[out]ctxInitialized AES-GCM context
[in]ivInitialization vector
[in]ivLenLength of the initialization vector in bytes. Must be non-zero. NIST recommends to use at most 96 bits for the IV, but the implementation does not enforce this.
Returns
Esc_NO_ERROR if everything works fine.

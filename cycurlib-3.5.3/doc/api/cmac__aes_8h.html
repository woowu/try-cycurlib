Detailed Description

CMAC implementation based on AES according to NIST SP 800-38B.

See also
NIST SP 800-38B

Key sizes of 128, 192 and 256 bits are allowed.

3832

Returns the MAC value calculated using CMAC in one step.

Parameters
[in]keyKey bytes array.
[in]keyLengthLength of the key array. Must be one of the following: 16 bytes (for CMAC-AES-128). 24 bytes (for CMAC-AES-192). 32 bytes (for CMAC-AES-256).
[in]messageThe input message for the AES-CMAC.
[in]messageLengthThe length of message in bytes. Does not need to match AES block boundaries.
[out]macBuffer to store CMAC value of macLength bytes. Can be the same as the message buffer.
[in]macLengthThe desired length of the resulting MAC, in bytes. This parameter can be used to truncate the MAC such that only the first bytes are copied to the output buffer. Must not be zero and must not be larger than EscCmacAes_MAX_MAC_LENGTH.
Returns
Esc_NO_ERROR if everything works fine.

This function resets the internal state such that a new MAC can be computed after EscCmacAes_Finish(). Calling this function after EscCmacAes_Init() is not required.

◆ EscEcc_ComputeSharedSecret()

Esc_ERROR EscEcc_ComputeSharedSecret ( Esc_UINT8  sharedSecret[],
Esc_UINT32 *  sharedSecretLen,
const Esc_UINT8  privKey[],
Esc_UINT32  privKeyLen,
const EscEcc_PublicKeyT *  pQB,
const EscEcc_CurveId  curveId 
)

This function computes the shared secret z based on ECDH.

  • P = dA * QB
  • Shared Secret: z = P->x
  • Private Key of A: dA (privKey)
  • Public Key of B: QB
Warning
This function does NOT validate the public key pQB! You can use EscEcc_PublicKeyValidation() to check if the public key is valid.
Parameters
[out]sharedSecretShared secret z.
[in,out]sharedSecretLenIn: Size of the sharedSecret buffer in bytes. EscEcc_KEY_BYTES_MAX can be used as the maximum size allowed by configuration. Out: Size of the shared secret in bytes
[in]privKeyPrivate key.
[in]privKeyLenLength of the private key in bytes. This must match the selected curve (curve.keySizeBytes).
[in]pQBPublic key of B.
[in]curveIdThe ID of the selected elliptic curve.
Returns
Esc_NO_ERROR if everything works fine.

◆ EscEcc_PublicKeyValidation()

Esc_ERROR EscEcc_PublicKeyValidation ( const EscEcc_PublicKeyT *  pQ,
const EscEcc_CurveId  curveId 
)

This function validates a public key with respect to the selected curve.

Elliptic Curve Public Key Validation according to ANSI X9.62-2005 Section A.4.2.

Check, that:

  • Q != (0)
  • Qx & Qy are properly represented elements in GF(p)
  • Q lies on elliptic curve defined by a & b

Note that a point multiplication with the base point order with a subsequent check for zero is not required for Brainpool or NIST prime curves.

Parameters
[in]pQPublic key point (affine).
[in]curveIdThe ID of the selected elliptic curve.
Returns
Esc_NO_ERROR if everything works fine or validation error

ECDSA Signature Generation according to ANSI X9.62-2005 Section 7.3.

Parameters
[out]signatureECC signature.
[in]msgHashHash of the message to be signed.
[in]msgHashLenLength of the hash in bytes. If the hash is shorter than the key size of the selected curve, the leftmost (most significant) bits are padded with zero. If longer, only the leftmost (most significant) bits of the hash are used. Note: The maximum hash length is currently limited to 64 bytes.
[in]privKeyPrivate key.
[in]privKeyLenLength of the private key in bytes. This must match the selected curve (curve.keySizeBytes).
[in]randomStateRandom state which may be required by your random implementation.

[in]curveIdThe ID of the selected elliptic curve.
Returns
Esc_NO_ERROR if everything works fine.

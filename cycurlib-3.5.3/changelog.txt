=== Version 3.5.3 =============================================================

- Bugfix:
    Fixed inconsistent data types for the nonce of the AES-EAX mode. The streaming
    interface (update function) used an UINT32 whereas the one-shot functions
    (encrypt/decrypt) used an UINT8.

=== Version 3.5.2 =============================================================

- Bugfix:
    Fixed a bug in the unwrap function of AES key wrapping: The authenticity check
    was skipped due to the bug. Therefore, the function returned no error if
    the cipher text was modified.

=== Version 3.5.1 =============================================================

- Bugfix:
    Fixed a bug in RSA key generation: The generated key is incorrect with a
    probability of approximately 1.37%. Despite the invalid key, the key generation
    function returns no error.

=== Version 3.5 ===============================================================

- Added new algorithms:
    * AES-EAX mode
    * Key wrapping for AES
    * KDF according to NIST SP800-56A
    * Miyaguchi-Preneel compression and key derivation according to SHE

- Improvements:
    * Faster ECC core operations
    * New fixed-base implementation to speed up ECDSA signature generation
    * Variable length of plaintext in ECIES implementation
    * Start() function for CMAC to avoid re-computation of schedule

- Removed legacy algorithms:
    * MD2
    * MD5
    * DES
    * ARC4

=== Version 3.4 ===============================================================

- Extended implementation of EdDSA to support variants in RFC 8032,
  i.e. pre-hashed EdDSA and the usage of a context.

- Changed X.509 parser to accept certificates with unknown attributes inside the
  distinguished names (i.e. subject/issuer). Multiple attributes of the same
  type are allowed as well. However, information about these attributes cannot
  be queried via the parser's API. New getter functions were added to support
  comparing an issuer with a subject.

- Removed EscDer_NextIntegerLongLe() function in the DER parser.
- Removed support for querying a second OU attribute in the X.509 parser.
  
=== Version 3.3 ===============================================================

- Added new algorithms:
    * Poly1305 one-time authenticator
    * ChaCha20_Poly1305 Authenticated Encryption with Associated Data (AEAD)
    * HMAC-SHA2-384, HMAC-SHA2-512

- Improved X.509/DER parser:
    * Support for two Organizational Unit (OU) attributes
    * Support for RDN attributes encoded as TeletexString, UniversalString, BMPString

=== Version 3.2 ===============================================================

- Added new algorithms:
    * SHA-3 hash function with digest sizes 224, 256, 384 and 512 bits
    * Ed25519 signature generation and verification with SHA2-512
    * ChaCha20 symmetric cipher

- Improved X.509 parser:
    * Added support for certificates signed with RSASSA-PSS.
    * Added support for the basicConstaints and keyUsage extensions.
      Certificates containing critical unknown/unsupported extensions are rejected.
    * Both RSA and ECDSA signatures are supported simultaneously at runtime now.
      Similarly, both RSA and ECC public keys are supported simultaneously at runtime now.
    * Parsing hybrid certificates is possible now (e.g. if the certificate contains an
      ECC public key, but is signed with RSA).
    * Support for all SHA-1 and SHA-2 hash functions.
    * Support for X509v1 and X509v2. Previously, only X509v3 was supported.

- Removed support for elliptic curve secp160r1

=== Version 3.1 ===============================================================

- Added time-sliced implementations of ECC/RSA operations

=== Version 3.0 ===============================================================

- AES:
    * Support for multiple key sizes at runtime. The maximum size can be configured at compile-time.
    * Moved decryption code to separate C file.
    * Changed ECB mode to support multiple blocks in a single call.
    * Improved speed of GCM implementation (variant without lookup tables).
    * Less restrictive implementation for AES-CTR (allow in-place processing and lengths which are not
      a multiple of the AES block size)
    * Support chunk-wise update of additional authenticated data in GCM
    * Separated key schedule computation from remaining data dependent initialization in AES-CCM
      (no need to re-compute key schedule any more if a new message is processed using the same key)

- RSA / PKCS#1:
    * Support for multiple key sizes at runtime. The maximum size can be configured at compile-time.
    * Use variable salt length in PKCS1-RSASSA-PSS.
    * Support for multiple hash algorithms at runtime in PKCS1-RSASSA-PSS.
    * Refactoring of key structures to be able to use generated keys directly.
    * Improved speed of CRT and key generation.

- ECC:
    * Support multiple elliptic curves at runtime. Support for each curve can be enabled or disabled at compile-time.
    * Added key pair generation (previous versions could only drive public keys from private keys)

- Support for truncation of digests / MACs    
- Switched to MISRA-C:2012
- Added header dependency generation to the Makefile
